# Security Policy

## Reporting a Vulnerability

We take the security of PyNeuraLogic seriously. If you discover a security vulnerability, please **do not open a public issue**. Instead, report it privately via email:

**lukaszahradnik96@seznam.cz**

Please include the following details in your report:

- A clear description of the vulnerability
- Steps to reproduce it (if applicable)
- The version(s) affected
- Any potential impact or exploit scenario

We will acknowledge your report within **5 business days** and work with you to understand and address the issue promptly.

## Supported Versions

We recommend using the latest published version of `neuralogic` from PyPI. Security patches are backported only to the most recent minor release.

| Version | Supported |
|---------|-----------|
| 0.8.x   | ✅ |
| < 0.8   | ❌ |

## Disclosure Policy

Once a fix has been developed and released, we will publish a security advisory on GitHub and credit the reporter (with their consent).

Thank you for helping keep PyNeuraLogic and its users safe.
