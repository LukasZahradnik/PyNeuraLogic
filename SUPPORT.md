# Support

## 📖 Documentation

The full documentation is available at [pyneuralogic.readthedocs.io](https://pyneuralogic.readthedocs.io/en/latest/). It includes tutorials, API references, and examples to help you get started.

## 🐛 Bug Reports & Feature Requests

If you've found a bug or have an idea for a new feature, please open an issue on GitHub:

- [Report a bug](https://github.com/LukasZahradnik/PyNeuraLogic/issues/new?assignees=&labels=bug&template=bug_report.yaml&title=%5B%F0%9F%90%9B+Bug+Report%5D%3A+)
- [Request a feature](https://github.com/LukasZahradnik/PyNeuraLogic/issues/new?assignees=&labels=enhancement&template=feature_request.yaml&title=%5B%E2%9C%A8+Feature+Request%5D%3A+)

Before opening, please check the [existing issues](https://github.com/LukasZahradnik/PyNeuraLogic/issues) to see if it's already been reported.

## 💬 Discussions

For questions, ideas, or general discussion, join our [GitHub Discussions](https://github.com/LukasZahradnik/PyNeuraLogic/discussions). This is the best place to:

- Ask "how do I do X?"
- Share projects built with PyNeuraLogic
- Propose new directions or get feedback

## 🤝 Contributing

If you'd like to contribute to the project, please read our [Contributing Guide](CONTRIBUTING.md). It covers the development workflow, testing, and pull request guidelines.
